DHCP Hatası: Bilgisayar IP Adresi Alamıyor

Bilgisayarınız ağa bağlanıyor ama internet çalışmıyor ve IP adresiniz 169.254.x.x görünüyor. Bu klasik bir DHCP sorunudur. Çoğu durumda birkaç komutla çözülür.

DHCP Nedir?

DHCP (Dynamic Host Configuration Protocol), modeminizin ağa bağlanan her cihaza otomatik olarak bir IP adresi, ağ geçidi ve DNS bilgisi atadığı protokoldür. DHCP çalışmazsa cihaz geçici bir "APIPA" adresi (169.254.x.x) alır ve internet erişimi sağlanamaz.

Sorunun Belirtileri

  • IP adresi 169.254.x.x formatında görünüyor
  • Ağ geçidi (default gateway) boş veya 0.0.0.0
  • WiFi bağlı görünüyor ama internet açılmıyor
  • "Tanımlanamayan ağ" veya "İnternet erişimi yok" uyarısı

Adım 1: Modemi Yeniden Başlatın

DHCP sorunlarının büyük çoğunluğu modem yeniden başlatılınca çözülür. Fişi çekin, 30 saniye bekleyin, takın.

Adım 2: IP Adresini Yenileyin

1 CMD'yi yönetici olarak açın (Başlat → cmd → Yönetici olarak çalıştır).
2 ipconfig /release yazın, Enter'a basın.
3 ipconfig /renew yazın, Enter'a basın.
4 ipconfig yazın — artık 192.168.x.x adresi görünmeli.

Adım 3: DHCP İstemci Servisini Kontrol Edin

Windows'ta DHCP servisinin çalıştığından emin olun:

  1. Win+R → services.msc yazın → Enter
  2. Listede DHCP İstemcisi (DHCP Client) servisini bulun
  3. Durum "Çalışıyor" olmalı. Değilse sağ tıklayın → Başlat
  4. Başlangıç türü "Otomatik" olarak ayarlayın

Adım 4: Ağ Bağdaştırıcısını Sıfırlayın

CMD yönetici modunda sırasıyla şu komutları çalıştırın:

  • netsh winsock reset
  • netsh int ip reset
  • ipconfig /flushdns

Ardından bilgisayarı yeniden başlatın.

Adım 5: IP Ayarını Otomatik Yapın

Ağ bağdaştırıcısının IP ayarı manuel yapılmışsa DHCP'den adres alamaz:

  1. Denetim Masası → Ağ Bağlantıları → WiFi veya Ethernet sağ tık → Özellikler
  2. İnternet Protokolü Sürüm 4 (TCP/IPv4) → Özellikler
  3. "IP adresini otomatik olarak al" seçin
  4. "DNS sunucu adresini otomatik olarak al" seçin
  5. Tamam → kapat

Adım 6: Modemde DHCP Açık mı?

Modem panelinde (192.168.1.1) LAN Ayarları → DHCP Sunucusu bölümünü kontrol edin. DHCP sunucusunun Etkin olduğundan emin olun. DHCP havuzu doluysa IP aralığını genişletin.

⚠️ Ağda birden fazla modem/router varsa yalnızca birinin DHCP sunucusu etkin olmalıdır. Diğerlerinde DHCP'yi kapatın, aksi hâlde IP çakışması ve dağıtım çakışması yaşanır.

Adım 7: Sürücüyü Yeniden Yükleyin

Ağ adaptörü sürücüsü bozulmuşsa DHCP isteği gönderilemez. Aygıt Yöneticisi'nde ağ adaptörünü kaldırın (sürücü dosyalarını silmeden) ve bilgisayarı yeniden başlatın — Windows sürücüyü otomatik yeniden yükler.
